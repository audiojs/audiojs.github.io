## P0 — correctness, live on npm

1. **`dynamics-multiband` sample-rate bug** — `multiband.js:32` passes `{fs}` to `dynamics-compressor`, which reads only `sampleRate` (`envelope.js:7`) → per-band attack/release ballistics always 44100 regardless of rate (crossover gets correct fs; compressors don't). Manifest passes `fs: ctx.sampleRate` into the same kernel → **engine/WAM hosting inherits it**. ~8% timing error at 48k, 54% at 96k. Invisible to tests (they run at 44100). Fix: pass `sampleRate` (or read both in envelope); add a ≠44100 ballistics test.
2. **`@audio/dynamics` umbrella serves stale atoms** — published deps pin `^0.1.0` for deesser/opto/fet/vca/varimu while latests are 0.2.x (0.x caret never crosses minor). Cold install confirmed: opto/fet resolve 0.1.1; standalone-umbrella deesser resolves 0.1.1 (no band mode, no per-block attack/release fix) while the umbrella README documents the superset. Invisible to release.mjs (content-clean — semver-range staleness is a class the sweep can't see). Fix: bump 5 ranges, patch-publish.
3. **`synth-poly` voice contract false for its own named examples** — poly.js claims "every @audio generator is a valid voice" with `(freq, opts)` shape; `drum.membrane`/`fm`/`modal` are opts-only → freq lands positionally → **silent wrong pitch/duration** (verified: 880 Hz note renders ~37 Hz), `wavetable` throws. Only tested with hand-wrapped osc. Same class: `stretch-*` batch entries treat `[L,R]` channel input as opts → factor undefined → silent default-stretch (shift throws properly via validateInput); `osc({...})` object-arg returns 1s of NaN silently (whole-NaN pipeline ran with zero errors in smoke test).
4. **WAM adapter contract gaps** — `flags:['restart']` entirely unhandled: `setParam('attack')` on the real compressor silently does nothing; `streaming:false` never refused (wrapping saturate-tube in a worklet = silently wrong output); `events.in` declared-but-dead; `ctx.layouts/transport` hardcoded undefined. All silent-wrong-behavior, not load errors.
5. **Kernel-vs-manifest default mismatches (filter repo)** — diode-ladder/korg35/oberheim `resonance` 0 (kernel) vs 0.5 (manifest); resonator `bw` 50 vs 100; spectral-tilt `slope` 0 vs −3. Direct users and engine users get different sound. (saturate/amp/eq/dynamics/denoise all verified exact.)

## P2 — consistency debt (the structural answer)

10. **sampleRate vs fs** — the ecosystem's most common option has two names, split by family: `fs` = filter/eq/weighting/auditory/saturate/amp/denoise/effect/reverb/spatial/tune/sinusoidal/synth/voice/measure/defeedback/pitch/beat/mir/spectral (loudness too, but defaults **48000** vs everyone's 44100); `sampleRate` = engine/manifest ctx/pcm-convert/shift/stretch — and **three-way inside dynamics** (sampleRate ×8, dual ×4, fs ×4; produced defect #1). Manifests uniformly bridge host `sampleRate`→kernel `fs` (100% consistent, documented nowhere). Decision needed: one kernel-level name (fs is the DSP-literature majority) + dual-read shim where breaking.
11. **Units fork per family**: time = ms (dynamics, chorus/flanger) vs seconds (denoise, most of effect, reverb) vs raw samples (all window/hop params); `depth` = seconds in vibrato vs 0–1 in its 5 modulation siblings; gain = dB (dynamics/eq) vs linear (effect, softclip, wiener floors mixed dB/linear per atom); `drive` 0–1 (effect-distortion) vs 0.5–10 raw (saturate); selectivity vocabulary ×5 (Q/q, bw Hz, resonance 0–1, order, slope dB/oct); hop/frame spelling ×4 (frameSize/hopSize, frameSize/hop, frame-seconds, blockSeconds; tempogram has two hop semantics in one signature).
12. **Mutation semantics unpredictable**: in-place = effect (23/26), reverb (all, family-unique among transforms), denoise filter-half (4), dynamics fs-half (3) vs new-array = dynamics core (11), denoise STFT half, stretch/shift, saturate. Not derivable from name or family. Three stream idioms (dynamics named `*Stream` {write,flush} / denoise dual-mode callable / state-on-caller's-params — the latter an anti-pattern instance: schroeder `params._c`, effect-delay `params.buffer`, crossfeed `params._coefs`) + effect has none.
13. **Substrate fragmentation**: `@audio/stft` consumed only by denoise family — shift/stretch/sinusoidal use `fourier-transform/stft`; onset + 8 spectral manifests + sinusoidal hand-roll framing (sinusoidal recomputes windows per call — perf smell). "Single STFT everywhere" is false as an org claim. `@audio/window` has **zero consumers** — `@audio/stft` itself hand-rolls Hann. Engine natively reimplements LUFS/centroid/flatness/crest/MFCC rather than hosting the atoms (two independent BS.1770 implementations in one org) while pitch/mir stats properly reuse atoms — two-tier design, invisible and undocumented. Engine still imports deprecated `a-weighting` (fn/spectrum.js:9, passes `sr` that its API discards) though `@audio/weighting-a.response(f,fs)` is the absorbed replacement.
14. **Manifest coverage uneven** (163 total): 100% stretch/saturate/amp; effect 24/26 but **effect-gain missing** (the CONTRACT's own worked example); eq 4/9 with lowshelf/highshelf/parametric missing despite same-shape manifested siblings (oversight); **loudness-lufs missing** — sole loudness outlier, flagship measurement never entered the contract system; shift 3/15 (12 atoms reachable only via umbrella auto-selector); weighting 0, auditory 0, measure 0 (exactly stat-shaped), resample 0 (blocked on just-landed §frames — in flight), sinusoidal 0, voice 0, pitch 1 (and that one is processor-shaped — the only non-{stat,compute} manifest among 26 analyzer manifests), beat 0.
